# জেনারেটিকস (Generics): **টাইপ সেফটি বজায় রেখে রিইউজেবল কোড তৈরির উপায়**

প্রোগ্রামিংয়ের ক্ষেত্রে আমরা সবসময় চাই এমন কোড লিখতে যা বারবার ব্যবহার করা যায়। কিন্তু স্ট্যাটিক টাইপ ল্যাঙ্গুয়েজগুলোতে (যেমন TypeScript বা Java) রিইউজেবিলিটি এবং টাইপ সেফটি একসাথে বজায় রাখা বেশ চ্যালেঞ্জিং। এই সমস্যার সবচেয়ে কার্যকর সমাধান হলো জেনারেটিকস। এই ব্লগে আমরা জানবো কীভাবে জেনারেটিকস ব্যবহার করে ডাটা স্ট্রাকচারের তোয়াক্কা না করে টাইপ সেফ রিইউজেবল কম্পোনেন্ট তৈরি করা যায়।

---

## **জেনারেটিকস আসলে কী?**

সহজ কথায়, জেনারেটিকস হলো এমন এক ধরণের টুল যা আপনাকে 'টাইপ' কে একটি ভেরিয়েবল বা প্যারামিটার হিসেবে ব্যবহার করার সুযোগ দেয়। অর্থাৎ, ফাংশন বা ক্লাস তৈরি করার সময় আপনি নির্দিষ্ট কোনো ডাটা টাইপ (যেমন string বা number) উল্লেখ না করে একটি প্লেসহোল্ডার ব্যবহার করবেন, যা রানটাইমে বা কল করার সময় নির্ধারিত হবে।

---

## **কেন জেনারেটিকস প্রয়োজন?**

জেনারেটিকস ছাড়া রিইউজেবল কোড লিখতে গেলে আমরা সাধারণত `any` টাইপ ব্যবহার করি। কিন্তু এতে টাইপ সেফটি নষ্ট হয়। নিচের উদাহরণটি দেখুন:

```tsx
function identity(arg: any): any {
  return arg;
}

let result = identity("Hello"); // এখানে result এর টাইপ 'any' হয়ে গেছে
```

এখানে `any` ব্যবহারের কারণে কম্পাইলার জানে না যে আউটপুটটি আসলে কী। ফলে রানটাইমে এরর হওয়ার সম্ভাবনা বেড়ে যায়। জেনারেটিকস এই সমস্যার সমাধান করে।

---

## **জেনারেটিকস কীভাবে কাজ করে?**

জেনারেটিকস ব্যবহারের জন্য আমরা সাধারণত `<T>` ব্যবহার করি। এখানে `T` একটি টাইপ ভেরিয়েবল।

### **১. রিইউজেবল ফাংশন তৈরি**

নিচের কোডটিতে আমরা একই ফাংশন দিয়ে ভিন্ন ভিন্ন ডাটা টাইপ হ্যান্ডেল করছি কিন্তু টাইপ সেফটি বজায় থাকছে:

```tsx
function logAndReturn<T>(arg: T): T {
  console.log(typeof arg);
  return arg;
}

let num = logAndReturn<number>(100); // টাইপ হবে number
let str = logAndReturn<string>("Generics"); // টাইপ হবে string
```

এখানে `T` ইউজার যা ইনপুট দিচ্ছে সেই টাইপটি ক্যাপচার করে নিচ্ছে। ফলে আপনি যদি ভুলবশত string রিটার্ন করা ফাংশনকে number হিসেবে ব্যবহার করতে চান, তবে আইডিই (IDE) আপনাকে তৎক্ষণাৎ ওয়ার্নিং দেবে।

---

### **ডাটা স্ট্রাকচারের সাথে জেনারেটিকস**

জেনারেটিকস সবচেয়ে বেশি কার্যকর যখন আমরা জটিল ডাটা স্ট্রাকচার যেমন অ্যারে (Array) বা অবজেক্ট নিয়ে কাজ করি।

### **২. টাইপ সেফ কালেকশন**

ধরুন একটি ফাংশন প্রয়োজন যা একটি অ্যারের প্রথম এলিমেন্টটি রিটার্ন করবে। জেনারেটিকস ব্যবহার করে এটি যেভাবে লেখা যায়:

```tsx
function getFirstElement<T>(arr: T[]): T {
  return arr[0];
}

const numbers = [1, 2, 3];
const firstNumber = getFirstElement(numbers); // কম্পাইলার জানে এটি একটি number

const names = ["Abir", "Rahim", "Karim"];
const firstName = getFirstElement(names); // কম্পাইলার জানে এটি একটি string
```

### **জেনারেটিকস ব্যবহারের সুবিধা**

১. **টাইপ সেফটি:** কোড কম্পাইল করার সময়ই ভুল ধরা পড়ে।

২. **কোড রিইউজেবিলিটি:** একই লজিক বারবার না লিখে ভিন্ন ভিন্ন ডাটা টাইপের জন্য ব্যবহার করা যায়।

৩. **ইন্টেলিসেন্স সাপোর্ট:** কোড লেখার সময় আপনার এডিটর সঠিক সাজেশন দিতে পারে কারণ সে ডাটা টাইপ সম্পর্কে নিশ্চিত।

৪. **ক্লিন কোড:** অপ্রয়োজনীয় টাইপ কাস্টিং বা ডুপ্লিকেট কোড কমাতে সাহায্য করে।

## **উপসংহার**

আধুনিক সফটওয়্যার ডেভেলপমেন্টে স্কেলেবল এবং মেইনটেনেবল কোড লেখার জন্য জেনারেটিকস অপরিহার্য। এটি আপনাকে এমন ফ্লেক্সিবল কম্পোনেন্ট এবং ফাংশন তৈরি করতে দেয় যা ডাটা স্ট্রাকচার যাই হোক না কেন, টাইপ সিস্টেমের কঠোরতা বজায় রাখে। আপনি যদি আপনার প্রজেক্টের কোড কোয়ালিটি উন্নত করতে চান, তবে আজই জেনারেটিকস প্র্যাকটিস শুরু করুন।
